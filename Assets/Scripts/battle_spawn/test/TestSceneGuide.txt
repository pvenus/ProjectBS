================================================================================
ProjectBS 신규 적 소환 시스템 테스트 씬 구성 가이드 (TestSceneGuide.txt)
================================================================================

본 문서는 새로 도입된 초기 적 소환 시스템(Spawn System)을 씬에서 구성하고 테스트하는 
단계별 가이드라인을 제공합니다.

--------------------------------------------------------------------------------
1. 테스트용 씬 구성 방법
--------------------------------------------------------------------------------
기존 BattleManager 등의 레거시 로직을 수정하지 않고 신규 스폰 메커니즘을 단독 테스트하기 
위해 새 씬을 만듭니다.

[단계 1] 새로운 빈 씬 생성
- Unity 에디터 상단 메뉴에서 File -> New Scene을 클릭하여 빈 씬(Basic 2D or Empty)을 생성합니다.
- 씬 이름을 "SpawnSystemTestScene"으로 저장합니다.

[단계 2] BattleSpawnManager 오브젝트 생성
- Hierarchy 뷰에서 우클릭 -> Create Empty를 클릭해 빈 오브젝트를 생성하고 이름을 "BattleSpawnManager"로 변경합니다.
- 해당 오브젝트에 'BattleSpawnManager' 컴포넌트를 추가합니다.
- (선택 사항) BattleSpawnManager의 'spawnChannel' 필드에는 수동으로 생성한 SpawnChannelSO 에셋을 할당할 수 있습니다. 
  (단, 에셋을 만들지 않아도 NewSpawnSystemTester를 통해 동적 테스트 데이터를 자동 생성해 실행할 수 있습니다.)

[단계 3] NewSpawnSystemTester 오브젝트 생성
- Hierarchy 뷰에서 우클릭 -> Create Empty를 클릭해 빈 오브젝트를 생성하고 이름을 "NewSpawnSystemTester"로 변경합니다.
- 해당 오브젝트에 'NewSpawnSystemTester' 컴포넌트를 추가합니다.
- Melee Monster SO, Ranged Monster SO, Boss Monster SO 필드에 기존 프로젝트의 
  CharacterSO 에셋들을 드래그 앤 드롭으로 할당합니다.
  (예: c:\_UnityProjects\ProjectBS\Assets\Resources\ CharacterSO 에셋 파일 매핑)
  (비워둘 경우 Resources 폴더 내부의 기본 NPC 에셋 검색을 시도합니다.)

[단계 4] 씬 앵커 배치 (수동 설정 시)
- Hierarchy 뷰에 빈 오브젝트를 만들고 'BattleSpawnAnchor' 컴포넌트를 부착합니다.
- 인스펙터 창에서 'anchorId' 필드에 "SpawnAnchor_Center"라고 기입합니다.
- 이 오브젝트의 위치(Transform Position)를 조정해 적들이 생성될 월드 기준점을 정합니다.
  (씬에 앵커가 없으면 NewSpawnSystemTester가 런타임에 (0,0,0)에 자동으로 앵커를 생성해 구동합니다.)

--------------------------------------------------------------------------------
2. 런타임 테스트 실행 방법
--------------------------------------------------------------------------------
[단계 1] Unity Play 모드 진입
- 에디터 상단의 Play(▶) 버튼을 누릅니다.
- 씬 시작 시 게임세션(GameSession) 등이 없으면 콘솔 창에 임시 생성되었다는 경고 로그와 함께 자동으로 구성됩니다.

[단계 2] 테스트 트리거 및 조작 단축키
- Play 모드 상태에서 다음 단축키를 눌러 소환 테스트를 발동합니다.
  * 숫자 1 키: 고정 웨이브 순차 실행 테스트 (Fixed Channel Test)
    - Wave 0 -> Wave 1 -> Wave 2를 순차 실행합니다.
  * 숫자 2 키: 무한 웨이브 루프 테스트 (Infinite Channel Test)
    - Wave 0 실행 후 Wave 1 -> Wave 2 -> Wave 1 -> Wave 2 순으로 무한 반복 실행합니다.
  * K 키: 활성화된 모든 적 즉사 시뮬레이션 (Kill All Active Enemies)
    - 현재 웨이브의 소환된 모든 적들에게 즉사 데미지를 적용해 사망 처리합니다.
    - 적들이 죽으면 스케줄러가 이를 즉시 감지하여 다음 웨이브 혹은 채널 완료로 이행합니다.

- NewSpawnSystemTester 컴포넌트의 인스펙터 콘솔 우클릭 컨텍스트 메뉴(Context Menu)를 통해서도 
  동일한 기능(Run Fixed Channel Test, Run Infinite Channel Test, Kill All Active Enemies)을 발동시킬 수 있습니다.

--------------------------------------------------------------------------------
3. 기획 데이터 에셋 생성 방법 (ScriptableObject)
--------------------------------------------------------------------------------
런타임 자동 생성 코드 외에, 프로젝트 기획 데이터 에셋으로 직접 소환 스케줄을 빌드하려면 
Project 뷰 우클릭 -> Create -> BS -> Spawn 메뉴군을 이용합니다.

- 스폰 유닛 연결
  - SpawnSquadSO group에는 spawnUnitKey 또는 spawnRole을 지정합니다.
  - 실제 CharacterSO 연결은 Spawner/BattleSpawnManager의 SpawnUnitBinding에서 지정합니다.
- [SpawnSquadSO] 생성: Create -> Battle Spawn -> Content -> Squad
  - formationPatternKind/config로 부대 기준점 배치를 설정합니다.
  - group 목록의 patternKind/config로 기준점 내부 몬스터 배치를 설정합니다.
  - 각 group에는 spawnUnitKey 또는 spawnRole을 입력합니다.
- [SpawnWaveSO] 생성: Create -> BS -> Spawn -> SpawnWave
  - 웨이브 내에 실행할 패턴 스케줄 리스트를 구성하고 startDelay와 nextWaveDelay를 지정합니다.
- [SpawnChannelSO] 생성: Create -> BS -> Spawn -> SpawnChannel
  - 소환 모드(Fixed / Infinite)를 설정하고 웨이브 목록을 순서대로 등록합니다.
