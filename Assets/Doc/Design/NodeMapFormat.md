# 노드맵 작성 룰

- formatId: `design.rule.node_map`
- version: `2`
- target: `Slay the Spire` 방식의 분기형 에피소드 노드맵
- scope: 스토리 에피소드 사이에 생성되는 랜덤 노드맵의 시각화 및 설계 기준

## 목적

메인 에피소드의 선택지를 중심 줄기로 사용하고, 그 사이에 랜덤 노드를 배치해 매번 다른 경로가 만들어지는 맵을 구성한다.

맵은 아래에서 위로 진행되며, 플레이어는 항상 연결된 길을 따라 다음 노드로 이동할 수 있어야 한다. 갈라진 메인 에피소드 라인은 중간에 임의로 합쳐지지 않고, 마지막 최종 에피소드에서 하나의 노드로 합류한다.

## 노드 종류

### 메인 에피소드 노드

- 스토리의 핵심 진행을 담당하는 고정 노드다.
- `Episode 1`, `Episode 2`처럼 에피소드 번호나 제목을 표시한다.
- 메인 에피소드에서 다음으로 파생되는 첫 랜덤 노드의 가로 개수는 해당 에피소드의 선택지 개수를 따른다.
- 선택지가 여러 개인 에피소드는 선택지별로 다른 메인 라인이나 랜덤 구간으로 갈라질 수 있다.

### 랜덤 노드

- 메인 에피소드 사이에 배치되는 변동 노드다.
- 전투, 이벤트, 보상, 휴식, 상점, 미확인 사건 등으로 바뀔 수 있다.
- 시각화에서는 `?` 또는 아이콘으로 표시한다.
- 실제 생성 시 노드의 종류와 개수는 고정값이 아니라 규칙에 따라 랜덤으로 정한다.

### 최종 에피소드 노드

- 챕터의 마지막 메인 에피소드다.
- 앞에서 갈라진 모든 경로가 이 노드로 합류한다.
- 최종 에피소드 직전까지 갈라진 라인은 가능한 한 독립적으로 유지하고, 마지막 연결에서 하나로 모은다.

## 기본 진행 방향

- 맵은 아래에서 위로 읽는다.
- 아래쪽 노드가 이전 진행, 위쪽 노드가 다음 진행이다.
- 연결선은 항상 아래 노드에서 위 노드로 이어진다.
- 모든 노드는 플레이어가 실제로 도달 가능한 경로 위에 있어야 한다.

## 메인 에피소드 이후 랜덤 구간

메인 에피소드 다음에 배치되는 랜덤 구간은 세로로 3개의 층을 가진다.

```text
다음 메인 에피소드
랜덤 3층
랜덤 2층
랜덤 1층
현재 메인 에피소드
```

- `랜덤 1층`은 현재 메인 에피소드에서 직접 파생되는 첫 번째 랜덤 층이다.
- `랜덤 2층`과 `랜덤 3층`은 이전 랜덤 노드에서 파생되는 층이다.
- 세 층의 가로 노드 개수는 고정하지 않는다.
- 단, 첫 번째 랜덤 층과 이후 랜덤 층은 서로 다른 기준으로 개수를 정한다.

## 가로 노드 개수 규칙

### 메인 에피소드에서 파생되는 첫 랜덤 층

- 첫 랜덤 층의 가로 노드 개수는 메인 에피소드의 선택지 개수를 따른다.
- 선택지가 1개면 첫 랜덤 층은 1개로 시작한다.
- 선택지가 2개면 첫 랜덤 층은 2개로 시작한다.
- 선택지가 3개면 첫 랜덤 층은 3개로 시작한다.
- 이 층은 플레이어가 고른 선택지의 방향성을 시각적으로 보여주는 역할을 한다.

### 랜덤 노드에서 파생되는 다음 랜덤 층

- 랜덤 노드에서 다음 층으로 이어질 때는 각 랜덤 노드가 1~3개의 다음 노드를 생성할 수 있다.
- 생성 개수는 고정하지 않고 랜덤으로 결정한다.
- 같은 층 안에서 노드가 너무 과하게 많아지면 가까운 노드를 합쳐도 된다.
- 단, 합쳐진 노드는 어느 이전 노드에서 도달할 수 있는지 연결선으로 분명히 보여야 한다.
- 한 층의 최종 노드 수는 미리 정해진 패턴이 아니라 생성 결과에 따라 달라진다.

## 연결 규칙

- 모든 랜덤 노드는 아래쪽 층의 노드 중 하나 이상과 연결되어야 한다.
- 최종 에피소드가 아닌 모든 노드는 위쪽 층의 노드 중 하나 이상과 연결되어야 한다.
- 아래에서 위로 올라갔을 때 끊어진 길이 없어야 한다.
- 노드가 여러 개로 갈라질 수는 있지만, 한 번 선택한 큰 분기 라인이 반대쪽 라인으로 무리하게 넘어가면 안 된다.
- 연결선은 가능한 한 가까운 위치의 노드끼리 잇는다.
- 경로가 교차하거나 겹쳐서 읽기 어려워지는 연결은 피한다.
- 랜덤 노드가 1개에서 여러 개로 늘어날 때는 자연스럽게 펼쳐지도록 연결한다.
- 여러 개에서 1개로 줄어들 때는 자연스럽게 모이도록 연결한다.

## 분기 유지와 최종 합류

- 메인 에피소드의 선택지로 갈라진 라인은 중간 랜덤 구간에서 임의로 합쳐지지 않는다.
- 같은 선택지에서 파생된 랜덤 노드끼리는 서로 이어질 수 있다.
- 다른 선택지에서 파생된 라인과 연결하려면 스토리상 합류 지점이 명확해야 한다.
- 챕터의 마지막 최종 에피소드에서는 모든 분기 라인이 하나의 노드로 합쳐진다.
- 최종 합류 직전에는 각 라인의 마지막 랜덤 노드들이 최종 에피소드 노드로 연결된다.

## 시각화 규칙

- 메인 에피소드 노드는 랜덤 노드보다 크고 눈에 띄게 표시한다.
- 랜덤 노드는 같은 크기와 형태로 통일한다.
- 서로 다른 메인 선택지에서 파생된 라인은 색상이나 위치로 구분할 수 있다.
- 같은 층의 노드들은 가로 간격을 균등하게 둔다.
- 노드와 연결선이 겹치거나 너무 가까워지지 않게 배치한다.
- 이미지 파일로 만들 때는 `SVG`를 우선 사용한다.
- 문서에서 보여줄 때는 `.md` 안에 이미지 링크를 넣어 원본 이미지를 유지한다.

## Chapter 1 적용 예시

Chapter 1의 구체적인 랜덤 노드 수는 고정하지 않는다. 대신 아래 흐름을 따른다.

```text
Episode 1
  |
Episode 2
  |-- Episode 3 계열 선택지
  |     랜덤 1층: Episode 3 계열 선택지 수 기준
  |     랜덤 2층: 각 랜덤 노드가 1~3개 생성
  |     랜덤 3층: 각 랜덤 노드가 1~3개 생성
  |
  |-- Episode 4 계열 선택지
        랜덤 1층: Episode 4 계열 선택지 수 기준
        랜덤 2층: 각 랜덤 노드가 1~3개 생성
        랜덤 3층: 각 랜덤 노드가 1~3개 생성

갈라진 라인들은 중간에 합쳐지지 않고,
마지막 Episode 6에서 하나의 최종 노드로 합류한다.
```
