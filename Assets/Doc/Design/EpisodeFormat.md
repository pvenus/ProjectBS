# 에피소드 작성 룰

- formatId: `design.rule.episode_format`
- version: `2`

## 목적

에피소드 스토리를 작성할 때 AI가 따라야 할 서사 규칙을 정리한다.

이 문서는 JSON 필드 포맷이 아니라, 에피소드의 흐름, 선택지, 전투 배치, 인물과 적 묘사 방식을 일관되게 만들기 위한 작성 기준이다.

## 기본 원칙

에피소드는 플레이어가 한 장면 안에서 상황을 이해하고, 선택을 통해 다음 흐름으로 넘어갈 수 있게 작성한다.

각 에피소드는 다음 요소를 분명히 가져야 한다.

- 지금 어떤 장소에 있는지
- 누가 등장하는지
- 무엇이 문제인지
- 플레이어가 무엇을 선택할 수 있는지
- 선택 결과로 어떤 흐름이 열리는지

스토리는 사건 설명만 나열하지 말고, 플레이어가 장면을 머릿속에 그릴 수 있도록 작성한다.

## 문서 관계와 필수 표기

이 문서는 `Assets/Doc/Design/StoryWritingRules.md`의 Episode 작성 규칙을 구체화한다.

두 문서를 함께 적용하며, 전투 표기는 다음 규칙을 예외 없이 따른다.

- 실제 전투로 진입하는 Episode에는 반드시 `[전투] 전투 제목`을 작성한다.
- `[전투]` 표기는 Chapter 요약 파일뿐 아니라 개별 Episode 파일에도 필수다.
- 전투 선택지 또는 전투 결과 문장이 있더라도 `[전투]` 표기를 생략하지 않는다.
- `[전투]` 표기가 있더라도 전투 선택지와 선택 결과 문장을 생략하지 않는다.
- 전투가 없는 조사, 대화, 이동 Episode에는 `[전투]`를 작성하지 않는다.
- 적의 흔적이나 감시 기척만 있고 실제로 싸우지 않는 장면은 전투로 분류하지 않는다.

전투 Episode의 최소 표기 구조는 다음과 같다.

```text
# 에피소드 제목

에피소드 기능 요약

장소, 사건, 단서 설명

[전투] 장소 또는 적과 목적이 드러나는 전투 제목

위험의 징후, 적의 외형, 장비, 표식, 공격 목적 설명

선택: 누구를 지키거나 어떤 적을 돌파하는지 드러나는 행동 문장

선택 결과로 전투가 시작되는 상황을 설명하는 문장
```

비전투 Episode의 최소 표기 구조는 다음과 같다.

```text
# 에피소드 제목

에피소드 기능 요약

장소, 인물, 단서 설명

선택: 조사하거나 이동할 대상을 드러내는 행동 문장

선택 결과로 다음 조사나 장소가 열리는 문장
```

## 선택지 룰

선택지는 모두 같은 기능을 가질 필요가 없다.

선택지에는 다음 종류가 섞일 수 있다.

- 단서를 조사하는 선택지
- 인물의 말을 더 듣는 선택지
- 다음 장소로 이동하는 선택지
- 동료 후보를 만나는 선택지
- 전투로 이어지는 선택지
- 전투 없이 조용히 위험을 피하거나 확인하는 선택지

선택지를 작성할 때는 버튼 문구만으로도 플레이어가 무엇을 하려는지 알 수 있어야 한다.

좋은 선택지 예시:

- `아이의 가족에게 직접 듣는다.`
- `유진의 흔적을 따라간다.`
- `지한과 함께 피 묻은 흔적을 확인한다.`
- `산채를 공격한다.`

피해야 할 선택지:

- `계속한다.`
- `확인한다.`
- `간다.`

단, 정말 단순한 연결 지문이면 짧은 선택지도 사용할 수 있다.

## 전투 선택지 룰

선택지 중에는 전투가 일어나는 선택지도 있고, 전투가 없는 선택지도 있다.

전투 선택지는 `[전투] 전투 제목`을 대신하지 않는다. 실제 전투가 시작되는 선택지가 하나라도 있다면 해당 전투 장면 앞에 `[전투]` 표기를 먼저 작성한다.

전투가 들어가는 에피소드는 대부분 마지막 부분에서 전투가 시작되도록 구성한다. 그래야 에피소드의 긴장감이 자연스럽게 쌓이고, 선택 후 전투로 넘어가는 흐름이 명확해진다.

전투가 필요한 경우:

- 마을이나 동료를 지켜야 할 때
- 추격자가 길을 막았을 때
- 적의 본거지를 돌파해야 할 때
- 아이나 단서를 빼앗기지 않아야 할 때
- 에피소드의 목적이 전투 진입일 때

전투가 없어야 더 좋은 경우:

- 단서를 처음 발견하는 조사 에피소드
- 인물의 가치관이나 동기를 보여주는 에피소드
- 분기 선택을 준비하는 에피소드
- 적의 흔적만 확인하고 아직 정면 충돌하지 않는 에피소드
- 위험을 피해서 지나가는 편이 더 자연스러운 장면

전투가 없는 에피소드에서도 긴장감은 유지할 수 있다. 이때는 적과 직접 싸우게 하지 말고, 흔적, 소리, 남겨진 물건, 도망친 자국, 감시의 기척 등을 통해 위협을 보여준다.

## 전투 배치 룰

전투가 있는 에피소드는 일반적으로 다음 흐름을 따른다.

1. 사건 또는 단서를 확인한다.
2. 등장인물의 판단이나 감정이 드러난다.
3. `[전투] 전투 제목`으로 실제 전투 장면임을 표시한다.
4. 적이나 위험이 모습을 드러낸다.
5. 적의 외형과 상태를 묘사한다.
6. 적이 어떤 방식으로 위협이 되는지 보여준다.
7. 플레이어 선택지로 전투에 진입한다.
8. 선택 결과 문장으로 전투 시작 상황을 정리한다.

`[전투]` 제목은 짧고 명확하게 작성하며 장소, 적, 목적 중 하나 이상을 포함한다.

좋은 예시:

```text
[전투] 폐광 입구의 관리병
[전투] 아이의 나무패를 지키는 뼈 인형
[전투] 산채 입구 돌파
```

피해야 할 예시:

```text
[전투] 싸움
[전투] 적 등장
[전투] 전투 1
```

전투를 갑자기 시작하지 않는다.

나쁜 예시:

```text
산적들이 나타났다.

선택: 싸운다.
```

좋은 예시:

```text
[전투] 숲길의 검은 천 추격자

숲길 끝에서 검은 옷자락들이 하나씩 모습을 드러냈다. 그들은 얼굴을 천으로 감싸고 있었고, 허리에는 붉은 실과 짐승 이빨을 꿴 끈을 매달고 있었다. 가장 앞의 사내는 키가 컸지만 어깨가 비정상적으로 말라 있었고, 손에는 갈고리가 달린 쇠사슬을 끌고 있었다.

선택: 길을 막아선 무리를 돌파한다.

서진은 주민들을 등 뒤에 두고, 검은 천의 무리 사이로 뛰어들었다.
```

## 적 묘사 룰

전투가 있다면 적들이 어떤 모습을 하고 있는지 세세하게 묘사한다.

적 묘사는 공격 방식보다 먼저 들어가는 것이 좋다. 플레이어가 먼저 알아야 하는 것은 "무엇이 나를 공격하는가"이기 때문이다.

적을 묘사할 때는 다음 순서를 우선한다.

1. 전체 실루엣
2. 키와 체형
3. 얼굴 또는 얼굴을 가린 방식
4. 옷의 길이, 색, 질감, 훼손 상태
5. 손과 팔, 발목 같은 신체 세부
6. 무기와 소지품
7. 장식, 표식, 부적, 문양
8. 그 적이 주는 전체 인상

예시:

```text
셋은 마른 체구에 등이 살짝 굽어 있었고, 하나는 다른 자들보다 머리 하나는 더 컸다. 모두 무릎 아래까지 내려오는 검은 겉옷을 입고 있었는데, 옷자락은 숲길에 끌려 해지고 진흙으로 굳어 있었다. 얼굴은 코와 입까지 천으로 감쌌고, 눈가에는 잠을 오래 자지 못한 사람처럼 어두운 그늘이 내려앉아 있었다.
```

장식만 나열하면 안 된다.

나쁜 예시:

```text
그들의 옷자락 안쪽에는 부적과 붉은 실이 있었다.
```

좋은 예시:

```text
가장 앞에 선 자는 키가 작고 말랐지만 어깨가 유난히 솟아 있었다. 목은 짧고 턱은 뾰족했으며, 코와 입을 감싼 검은 천 아래로 누렇게 마른 피부가 조금 드러났다. 낡은 겉옷의 소매는 붕대로 여러 번 감겨 손등 절반을 덮고 있었고, 허리에는 붉은 실로 묶은 부적 꾸러미가 매달려 있었다.
```

## 적의 공격 묘사 룰

적의 공격 방식도 필요하지만, 외형 묘사보다 길어지면 안 된다.

공격 묘사는 다음 질문에 답하면 충분하다.

- 적이 멀리서 공격하는가, 가까이 붙는가?
- 누구를 노리는가?
- 무엇 때문에 위험한가?
- 플레이어가 어떤 상황을 막아야 하는가?

예시:

```text
앞줄의 무리들은 방패 대신 뜯어낸 문짝을 들고 주민들을 밀어붙였고, 뒤쪽의 자들은 돌과 짧은 투창으로 도망치는 길을 끊었다.
```

공격 방식만 길게 쓰면 전투 기획서처럼 보일 수 있으므로 주의한다.

## 동물과 괴이한 대상 묘사 룰

동물이나 괴이한 대상도 사람처럼 구체적으로 묘사한다.

동물을 묘사할 때는 다음을 포함한다.

- 털의 상태
- 눈, 이빨, 갈비뼈, 목덜미 같은 신체 특징
- 굶주림, 상처, 목줄 자국 같은 상태
- 사람이 조종했거나 길들인 흔적
- 부적, 붉은 천, 노끈 같은 장식

예시:

```text
산늑대와 멧개의 털은 군데군데 빠져 있었고, 등에는 마른 진흙과 붉은 천 조각이 엉겨 붙어 있었다. 목덜미에는 오래된 밧줄 자국이 패여 있었으며, 몇 마리는 이빨 사이에 작은 쇠고리 같은 것이 끼워져 있었다.
```

동물이 그냥 "굶주린 짐승"으로만 나오면 장면이 약해진다.

## 조사 에피소드 룰

조사 에피소드에는 전투를 억지로 넣지 않는다.

조사 에피소드에서는 다음 요소로 긴장감을 만든다.

- 누군가 일부러 지운 발자국
- 이상하게 남겨진 부적이나 천 조각
- 정찰대가 머문 흔적
- 피가 묻은 옷감
- 아이가 남긴 말이나 그림
- 주변에 있지만 아직 싸우지 않는 짐승이나 적의 기척

조사 에피소드의 선택지는 "싸운다"가 아니라 "확인한다", "따라간다", "살핀다", "피해 간다" 쪽이 자연스럽다.

예시:

```text
선택: 유진과 함께 지워진 흔적을 확인한다.

서진은 유진이 지워진 길을 다시 읽어낼 수 있도록 주변의 풀과 흙을 조심스럽게 살폈다.
```

## 동료 등장 에피소드 룰

동료가 등장하는 에피소드에서는 전투보다 동료의 역할이 먼저 보여야 한다.

지한 같은 치료형 인물은 다음 특징을 드러낸다.

- 다친 사람을 살리려는 태도
- 쉽게 포기하지 않는 신념
- 약상자, 붕대, 부목 같은 도구
- 조용하지만 단단한 말투

유진 같은 추적형 인물은 다음 특징을 드러낸다.

- 발자국과 길을 읽는 능력
- 숲과 산길에 익숙한 태도
- 활과 시야 확보
- 냉정하지만 경계심 있는 말투

동료 합류 선택지는 동료가 왜 필요한지 분명히 보여준 뒤 배치한다.

## 결과 문장 룰

선택 후 결과 문장은 너무 길게 쓰지 않는다.

결과 문장은 플레이어의 선택이 어떤 흐름을 만들었는지 정리한다.

예시:

```text
서진은 붉은 천 인형과 아이의 마지막 말을 따라 숲 안쪽을 조사하기로 했다.
```

전투 결과 문장은 전투가 시작되는 상황까지만 정리한다.

예시:

```text
서진은 주민들을 등 뒤에 두고, 검은 천의 무리 사이로 뛰어들었다.
```

## 스토리 지문 기획 룰

에피소드 본문은 나중에 1차 기획 데이터로 변환될 수 있어야 한다.

1차 기획 데이터는 정식 스크립트가 아니라 시놉시스 수준의 방향성이다.

지문에는 다음 정보가 자연스럽게 드러나는 것이 좋다.

- 플레이어가 지금 알아야 하는 사실
- 아직 감춰야 하는 사실
- 장면의 감정 톤
- 공간의 성격
- 위협의 정체 또는 오해
- 다음 선택으로 이어지는 이유

단, 지문 안에 JSON 태그나 전투 수치를 직접 쓰지 않는다.

예시:

```text
검은 천을 두른 무리들은 산적처럼 보였지만, 허리마다 같은 붉은 인형을 매달고 있었다.
```

이 문장은 1차 기획 데이터에서 다음처럼 해석될 수 있다.

```text
story.scriptSynopsis: 검은 천 무리는 단순 산적이 아닐 수 있다는 인상을 준다.
story.mustShow: 검은 천, 붉은 인형
story.mustHide: 산귀의 정체
monster: black_cloth_raiders, early_easy, basic_melee
battle: urgent_rescue, front_pressure_with_light_flank
```

## 보상 지문 룰

에피소드 지문은 보상을 설명할 수 있지만, 최종 보상 수치를 직접 고정하지 않는다.

현재 보상 기준은 골드만 사용한다.

참조 문서:

```text
Assets/character_concepts/game_prompt_guide/story/EpisodePlanningCreateGuide.md
Assets/character_concepts/game_prompt_guide/story/RewardPlanningGuide.md
```

좋은 방향:

```text
촌장은 아이를 지켜낸 서진에게 얼마 되지 않는 돈주머니를 건넸다.
```

피해야 할 방향:

```text
서진은 정확히 143골드를 받았다.
```

정확한 골드 수량은 보상 기획 단계에서 정한다.

## 에피소드 마무리 룰

에피소드의 마지막은 다음 중 하나로 닫는다.

- 다음 장소로 향하는 결정
- 다음 분기로 이어지는 단서
- 동료 합류
- 전투 진입
- 더 큰 위협의 암시

마지막 문장은 다음 에피소드를 궁금하게 만들어야 한다.

다만 조사 에피소드에서는 전투로 닫지 않아도 된다. 대신 "이 흔적이 어디로 이어지는지 확인해야 한다"처럼 다음 행동을 분명히 한다.

## 체크리스트

에피소드를 작성한 뒤 아래 항목을 확인한다.

- 이 에피소드가 조사, 분기, 동료 합류, 전투 중 어떤 역할인지 분명한가?
- 실제 전투가 있다면 `[전투] 전투 제목`이 반드시 있는가?
- `[전투]` 제목에 장소, 적, 목적 중 하나 이상이 드러나는가?
- 전투가 없는데 `[전투]` 표기를 잘못 넣지 않았는가?
- 플레이어가 알아야 하는 사실과 감춰야 하는 사실이 분리되어 있는가?
- 선택지가 서로 다른 행동을 제안하는가?
- 선택 결과가 보상, 분기, 전투, 동료 합류 중 무엇으로 이어지는지 알 수 있는가?
- 전투가 있다면 마지막에 자연스럽게 배치되었는가?
- 전투가 없는 에피소드에 억지 전투가 들어가지 않았는가?
- 적, 동물, 대상의 전체 실루엣이 먼저 보이는가?
- 장식이나 부적만 나열하고 몸집, 얼굴, 옷차림을 빼먹지 않았는가?
- 공격 방식이 외형 묘사보다 길어지지 않았는가?
- 조사 에피소드에서는 흔적과 기척만으로 긴장감을 만들었는가?
- 마지막 문장이 다음 행동으로 자연스럽게 이어지는가?
