# 스토리 제작 필수 요소

- formatId: `design.rule.story_required_elements`
- version: `5`

## 목적

스토리 파일을 바탕으로 Episode를 만들기 전에 최소한으로 정해야 하는 요소를 정리한다.

스토리 본문은 별도 md 파일로 제공한다고 가정한다. 이 문서는 해당 스토리 파일을 어디에서 읽을지, 그리고 Episode 제작에 필요한 구조 정보를 무엇으로 볼지 정리한다.

Episode를 만들 때 작성 규칙은 항상 `Assets/Doc/Design/StoryWritingRules.md`를 참조한다.

Chapter 스토리 경로는 입력받지 않는다. 대신 `생성할 Act 번호`와 `생성할 Chapter 번호`를 두 자리 숫자로 맞춰 다음 형식으로 자동 조합하고, Episode를 만들 때 해당 경로의 스토리 파일을 반드시 참조한다.

```text
Assets/Doc/Story/Act{Act번호}/Chapter{Chapter번호}
```

예시:

```text
생성할 Act 번호: 1
생성할 Chapter 번호: 1
자동 Chapter 스토리 경로: Assets/Doc/Story/Act01/Chapter01
```

자동 조합한 Chapter 폴더 안의 `Chapter_XX.md`를 Chapter 원본 스토리로 사용한다.

## 캐릭터 이름 사용 규칙

- `새로 등장하는 캐릭터 이름`이 입력되거나 원본 스토리에 명시된 경우에만 Episode에서 해당 이름을 사용한다.
- 캐릭터 이름이 없다면 임의의 이름을 만들거나 이전 Episode의 캐릭터 이름을 가져와 사용하지 않는다. 필요한 경우 이름 대신 직업, 역할, 관계 등의 일반적인 표현으로 지칭한다.
- 각 Episode의 주인공은 서로 연속되는 동일 인물로 간주하지 않는다. 원본 스토리나 입력 사항에 연결 관계가 명시된 경우에만 같은 인물로 취급한다.

## 필수 요소

새 스토리를 만들 때는 아래 항목을 먼저 정한다.

- 생성할 Act 번호
- 생성할 Chapter 번호
- 해당 Chapter의 Episode 수
- 분기가 있는 Episode
- 분기의 수
- 분기마다 달라지는 내용
- 분기가 다시 합쳐지는 지점
- 새로 등장하는 캐릭터 이름
- 새로 등장하는 캐릭터의 직업 또는 역할

## 분기 입력 해석 규칙

`분기가 있는 Episode`는 플레이어가 선택하는 Episode가 아니라, 선택 결과에 따라 **처음으로 갈라진 Episode 파일들이 시작되는 번호**를 뜻한다.

예를 들어 `분기가 있는 Episode: 2`, `분기 수: 3`이라면 다음 구조로 작성한다.

```text
01_episode1.md
  ├─ 선택 1 → 02-1_episode2-1.md
  ├─ 선택 2 → 02-2_episode2-2.md
  └─ 선택 3 → 02-3_episode2-3.md
```

- 분기 직전의 공통 Episode에는 분기 수와 동일한 개수의 진입 선택지가 반드시 있어야 한다.
- 각 진입 선택지는 서로 다른 분기 파일 하나에 일대일로 대응해야 한다.
- 선택지 문장만 읽어도 어떤 인물, 단서, 보상 또는 장소의 분기로 들어가는지 구별할 수 있어야 한다.
- 분기 파일만 만들고 직전 Episode에 진입 선택지를 만들지 않은 구조는 미완성으로 간주한다.
- Episode 2가 분기 시작점이면 Episode 1의 마지막 진행 선택지가 분기 선택지 역할을 한다. 앞선 전투 선택지가 있다면 전투 결과 뒤에 분기 선택 장면을 별도로 작성한다.

## 선택지 수 규칙

선택지는 반드시 분기를 만들 필요는 없다. 서로 다른 행동을 선택한 뒤 같은 Episode나 같은 다음 Episode로 이어져도 된다.

다만 하나의 결정 지점에 `선택:`이 있다면 실제 대안을 최소 2개 제공해야 한다.

- 선택지가 하나뿐인 결정 지점은 만들지 않는다.
- 가능한 행동이 하나뿐이면 `선택:`으로 표시하지 않고 일반 진행 문장으로 작성한다.
- 같은 목적지로 이어지는 선택지라도 접근 방식, 태도, 위험 부담, 조사 대상 또는 보상이 달라야 한다.
- 형식만 다른 사실상 동일한 행동을 여러 선택지로 복제하지 않는다.
- 이 규칙은 분기 생성 여부와 관계없이 모든 Episode에 적용한다.

## 스토리 본문 독립성 규칙

스토리 본문은 해당 문장만 읽어도 의미를 이해할 수 있어야 한다. 파일 구조나 제작 입력을 알아야 이해되는 메타 표현을 본문에 사용하지 않는다.

- 본문, 대사, 선택지, 선택 결과에서는 `Act`, `Chapter`, `Episode`, `분기`, `난이도`, `보상`, `이전 경로`, `각 Chapter에서` 같은 제작 용어를 사용하지 않는다.
- 앞선 이야기를 언급할 때는 구조 번호 대신 구체적인 장소, 사건, 인물의 역할 또는 물건 이름을 쓴다.
- 경로에 따라 달라지는 내용을 언급할 때는 실제로 소지한 물건이나 확인한 사건을 직접 서술한다.
- 독자가 이전 파일이나 기획 문서를 확인해야 뜻을 알 수 있는 표현은 사용하지 않는다.
- Episode 첫 문단의 기획용 기능 요약과 Chapter 요약 문서에서는 구조 설명을 위해 제작 용어를 사용할 수 있다. 단, 그 뒤의 플레이어용 스토리 본문에는 이어 쓰지 않는다.

나쁜 예:

```text
각 Chapter에서 얻은 단서를 펼쳤다.
분기에서 얻은 보상을 제단에 올렸다.
이전 Episode의 선택에 따라 방울이 달라졌다.
```

좋은 예:

```text
청운촌의 붉은 천 인형, 금광의 가짜 부적, 잊힌 신당의 제문 조각을 바위 위에 펼쳤다.
조사단은 산길에서 되찾은 나무패를 제단 홈에 올렸다.
품속의 은방울이 아이의 울음과 닮은 소리를 냈다.
```

## 간단 입력 양식

```text
생성할 Act 번호:
생성할 Chapter 번호:

Episode 수:
분기 있는 Episode:
분기 수:
분기별 차이:
분기 합류 지점:

새 캐릭터 이름:
캐릭터 직업/역할:
특이사항:
```

`새 캐릭터 이름`을 비워 두면 이름 없는 캐릭터로 작성하며, 다른 Episode의 주인공 이름을 이어서 사용하지 않는다.
