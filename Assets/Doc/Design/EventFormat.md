# 이벤트 스토리 작성 룰

- formatId: `design.rule.event_format`
- version: `1`

## 목적

메인 에피소드 사이에 배치되는 단독 이벤트 스토리의 작성 기준을 정리한다.

이벤트는 짧은 사건 안에서 상황을 제시하고, 플레이어가 2개 또는 3개의 행동 중 하나를 선택하게 한다. 선택 결과는 이야기의 전개와 직접 연결되어야 하며, 전투만 반복하지 않고 회복, 피해, 보상, 후속 사건 등 다양한 결과를 제공한다.

## 에피소드와의 차이

이벤트 파일은 에피소드 요약 문서처럼 맨 위에 줄거리 요약, 삽입 위치, 목적, 보상 목록을 먼저 적지 않는다.

제목 다음 줄부터 바로 장소와 상황을 묘사하며 스토리를 시작한다.

좋은 시작:

```text
# 길을 막은 쓰러진 장승

해가 기울 무렵, 산길을 걷던 일행은 길 한가운데 쓰러진 장승 하나를 발견했다.
```

피해야 할 시작:

```text
# 길을 막은 쓰러진 장승

산길에서 언제든 발생하는 독립 이벤트.
요약: 장승을 고치거나 지나치는 이벤트다.
보상: 골드 또는 유물.
```

이벤트는 메인 스토리의 핵심 단서나 특정 챕터의 진행 상태를 반드시 요구하지 않는다. 특정 인물의 합류 여부에 의존해야 한다면 범용 이벤트가 아니라 별도의 조건부 이벤트로 분류한다.

## 기본 구성

하나의 이벤트는 다음 순서로 작성한다.

1. 장소와 현재 상황을 보여준다.
2. 플레이어가 개입할 문제나 유혹을 제시한다.
3. 선택에 필요한 위험, 비용, 기대 가능성을 문장 안에서 암시한다.
4. 서로 다른 행동을 뜻하는 선택지 2개 또는 3개를 제시한다.
5. 각 선택지 바로 아래에 결과 태그와 그 결과에 맞는 스토리 지문을 작성한다.
6. 각 선택 결과는 그 자리에서 끝나거나 후속 사건으로 이어진다. 특수 이벤트로 이어질 때는 연계 스토리를 반드시 같은 파일에 작성한다.

## 선택지 개수와 표기

선택지는 반드시 2개 또는 3개로 구성한다.

선택지 문구는 `선택 1:`, `선택 2:`, `선택 3:` 형식으로 적는다.

```text
선택 1: 무너진 담을 직접 치우고 안쪽을 확인한다.
선택 2: 마을 사람을 불러 함께 담을 치운다.
선택 3: 위험을 감수하지 않고 길을 떠난다.
```

버튼 문구만 읽어도 플레이어가 무엇을 하려는지 알 수 있어야 한다.

좋은 선택지:

- `상처 입은 나그네에게 약초를 나누어 준다.`
- `돈주머니를 관아에 맡기지 않고 주인을 직접 찾는다.`
- `석상에 손을 넣어 안쪽의 물건을 꺼낸다.`

피해야 할 선택지:

- `도와준다.`
- `확인한다.`
- `그만둔다.`

선택지들은 같은 행동의 표현만 바꾼 것이 아니라 서로 다른 판단이어야 한다. 적극적으로 개입하기, 대가를 요구하기, 위험을 피하기, 다른 방법을 찾기처럼 행동과 태도가 구분되어야 한다.

## 선택 결과 비공개 원칙

플레이어가 선택하기 전에는 선택으로 발생할 시스템 결과를 명확히 고지하지 않는다.

선택지 문구에 `전투 발생`, `체력 회복`, `피해를 받는다`, `유물 획득`, `골드 획득`, `특수 이벤트 추가`, `전직 가능` 같은 결과 설명이나 결과 태그를 붙이지 않는다. 성공 여부, 정확한 보상량, 숨겨진 위험도 미리 확정해서 알려주지 않는다.

피해야 할 표기:

```text
선택 1: 상인을 도와준다. (전투 후 유물 획득)
선택 2: 상인을 외면한다. (아무 일도 일어나지 않음)
```

사용할 표기:

```text
선택 1: 쓰러진 수레를 일으킬 수 있도록 상인을 도와준다.
선택 2: 수상한 상인에게 관여하지 않고 지나간다.
```

다만 행동 자체에 결과가 자연스럽게 포함되어 있다면 플레이어가 보상의 종류를 추측할 수 있어도 괜찮다.

```text
선택 1: 떨어진 목걸이를 장터의 상점에 판다.
선택 2: 목걸이에 새겨진 문양을 단서로 주인을 찾는다.
선택 3: 목걸이를 자신의 물건으로 간직한다.
```

이 경우 1번에서 골드를 얻거나 3번에서 목걸이 유물을 얻을 수 있다고 예상하는 것은 자연스럽다. 선택지를 억지로 모호하게 만들어 이러한 상식적인 추측까지 막지 않는다.

결과 태그는 작성 문서에서 각 선택의 결과를 구분하기 위해 사용하지만, 플레이어가 선택하기 전에 노출되는 선택지 정보로 취급하지 않는다. 결과 태그와 결과 지문은 해당 선택을 확정한 뒤 표시되는 내용이다.

## 사용 가능한 결과

각 선택에는 아래 결과 중 1개 또는 2개의 조합이 발생한다.

- `[전투]`
- `[체력 회복]`
- `[피해]`
- `[유물 보상]`
- `[골드 획득]`
- `[특수 이벤트 추가]`
- `[전직 이벤트]`
- `[아무 일도 일어나지 않음]`

결과가 1개라면 태그를 하나만 사용한다.

```text
선택 1: 따뜻한 약차를 마시고 잠시 쉬어 간다.

[체력 회복]

노인은 가장 상태가 좋은 약초를 골라 찻주전자에 넣었다. 따뜻한 기운이 몸에 퍼지자 굳었던 팔다리가 가벼워졌다.
```

결과가 2개 조합이라면 태그를 연속으로 적은 뒤, 두 결과가 모두 일어난 이유를 하나의 스토리 흐름으로 설명한다.

```text
선택 2: 연기가 새어 나오는 굴 안으로 들어간다.

[피해]
[유물 보상]

뜨거운 증기가 팔과 얼굴을 덮쳤지만, 일행은 물러서지 않고 바위틈에 박힌 청동패를 꺼냈다. 화상은 남았으나 청동패에는 오래된 보호 문양이 온전히 새겨져 있었다.
```

결과 태그만 적고 끝내지 않는다. 반드시 태그 뒤에 선택 행동이 어떻게 해당 결과로 이어졌는지 보여주는 스토리 지문을 작성한다.

## 결과별 작성 기준

### `[전투]`

선택 때문에 적대자가 공격하거나 위험한 존재와 충돌할 때 사용한다.

전투가 시작되는 이유, 적의 모습, 싸워야 하는 목적을 지문에 포함한다. 전투 승리 뒤 추가 결과가 있다면 두 번째 태그로 표시한다.

```text
[전투]
[유물 보상]

상자를 들어 올리자 돌 틈에 숨어 있던 독충 떼가 쏟아졌다. 독충을 물리킨 뒤 상자를 열면 안쪽에서 작은 옥 장식이 발견된다.
```

`[전투]`는 가능한 결과 중 하나일 뿐이며 모든 이벤트에 넣지 않는다.

### `[체력 회복]`

휴식, 음식, 치료, 온천, 선행에 대한 답례처럼 회복의 원인이 서사 안에 있을 때 사용한다.

대가 없이 회복할 수도 있고, 골드 지불이나 다른 기회의 포기처럼 선택 자체가 비용이 될 수도 있다.

### `[피해]`

함정, 무리한 행동, 위험한 거래, 독이나 저주 등으로 플레이어가 손해를 입을 때 사용한다.

피해는 단순한 벌일 수도 있지만 `[유물 보상]`, `[골드 획득]`, `[특수 이벤트 추가]`, `[전직 이벤트]`와 조합해 위험을 감수할 이유를 만들 수 있다.

### `[유물 보상]`

희귀한 물건, 신비한 장신구, 특별한 도구처럼 유물을 획득할 때 사용한다.

유물은 사건 속에 원래 존재했거나 인물에게 답례로 받아야 한다. 이유 없이 바닥에서 나타나거나 결과 지문 없이 지급하지 않는다.

### `[골드 획득]`

의뢰 보수, 물건 판매, 사례금, 숨겨진 돈의 발견 등으로 골드를 얻을 때 사용한다.

선행의 대가일 수도 있고, 도덕적으로 불편한 선택의 이익일 수도 있다. 획득 경위를 지문에서 분명히 한다.

### `[특수 이벤트 추가]`

현재 사건에서 얻은 약속, 표식, 소문, 물건 또는 인연이 해당 이벤트와 연계된 후속 스토리를 열 때 사용한다.

연계되는 후속 스토리는 새로운 파일로 분리하지 않는다. 최초 이벤트와 같은 파일의 아래쪽에 이어서 작성하고, 각각 별도의 `# 제목`을 붙여 구분한다.

한 파일에 최초 이벤트와 연계 이벤트가 함께 들어가는 기본 구조는 다음과 같다.

```text
# 검은 가시의 표식

산길에서 검은 가시에 둘러싸인 나그네를 발견했다.

선택 1: 가시덤불을 헤치고 나그네를 구한다.

[피해]
[특수 이벤트 추가]

검은 가시에 손을 찔린 순간 손등에 초승달 모양의 자국이 남았다. 상처는 쓰라렸지만, 떠나려던 무당은 그 표식을 알아보고 다음 보름에 북쪽 신당으로 오라는 말만 남겼다.

선택 2: 위험한 가시덤불에 관여하지 않고 떠난다.

[아무 일도 일어나지 않음]

일행은 나그네의 외침을 뒤로하고 산길을 떠났다. 검은 가시는 곧 빈 길을 덮어 그곳에 누가 있었는지조차 감추었다.

# 초승달이 뜨는 신당

다음 보름, 손등의 표식이 희미하게 빛나기 시작했다. 일행이 무당이 알려준 북쪽 신당에 도착하자 닫혀 있던 돌문이 천천히 열렸다.

선택 1: 표식이 가리키는 신당 안쪽으로 들어간다.

[전직 이벤트]

돌문 안쪽에서 기다리던 무당은 검은 가시를 견딘 사람만 치를 수 있는 시험에 관해 설명했다.
```

`[특수 이벤트 추가]` 결과 지문에는 후속 스토리가 열리는 이유와 연결 단서를 남긴다. 장소, 인물의 약속, 특정 물건, 표식, 시간 조건처럼 플레이어가 다음 사건을 이해하는 데 필요한 정보가 있어야 한다. 다만 후속 스토리에서 벌어질 결과까지 미리 공개하지 않는다.

후속 스토리의 `# 제목`은 최초 이벤트의 제목과 구분되는 고유한 이름을 사용한다. 연계가 여러 단계라면 같은 파일 아래에 발생 순서대로 `# 제목`을 추가한다.

```text
# 최초 이벤트 제목

...

# 첫 번째 연계 이벤트 제목

...

# 두 번째 연계 이벤트 제목

...
```

후속 스토리 역시 일반 이벤트와 동일하게 다음 규칙을 따른다.

- 요약 없이 `# 제목` 다음부터 바로 스토리를 시작한다.
- 선택지는 2개 또는 3개로 구성한다.
- 각 선택에는 결과 태그 1개 또는 2개를 사용한다.
- 선택 전에 결과와 보상을 직접 고지하지 않는다.
- 선택 결과에 맞는 스토리 지문을 작성한다.

같은 파일에 후속 스토리가 작성되어 있지 않다면 `[특수 이벤트 추가]` 태그를 사용할 수 없다. 나중에 작성할 예정이라는 메모만 남기거나 존재하지 않는 별도 파일을 가리키지 않는다.

### `[전직 이벤트]`

새로운 직업이나 전직 가능성을 여는 스승, 시험, 자격, 계보를 발견했을 때 사용한다.

즉시 전직이 확정되는지, 전직을 위한 후속 시험이 열리는지는 결과 지문에서 구분한다. 일반적인 보상보다 드물게 사용하며, 선택 전 지문에 특별한 기회임을 짐작할 단서를 제공한다.

### `[아무 일도 일어나지 않음]`

플레이어가 개입하지 않거나, 시도했지만 실질적인 변화나 보상을 얻지 못했을 때 사용한다.

이 결과도 한두 문장의 스토리 지문으로 마무리한다. 태그만 표시하거나 아무 설명 없이 이벤트를 끝내지 않는다.

```text
[아무 일도 일어나지 않음]

일행은 수상한 소리를 뒤로하고 원래 걷던 길로 돌아갔다. 한동안 방울 소리가 따라왔지만 고개를 넘자 더는 들리지 않았다.
```

`[아무 일도 일어나지 않음]`은 다른 결과와 조합하지 않는다. 다른 효과가 발생했다면 이미 아무 일도 일어나지 않은 결과가 아니기 때문이다.

## 선택과 결과의 연결

선택 결과는 플레이어의 행동에서 자연스럽게 이어져야 한다.

- 위험한 장소에 직접 들어가면 피해나 전투 가능성이 생긴다.
- 시간을 들여 사람을 도우면 회복, 사례금, 유물 또는 후속 인연을 얻을 수 있다.
- 물건을 팔거나 보수를 요구하면 골드를 얻을 수 있다.
- 수상한 제안이나 시험을 받아들이면 전직 이벤트나 특수 이벤트가 열릴 수 있다.
- 위험을 피하고 떠나면 아무 일도 일어나지 않을 수 있다.

선택 전에 결과를 정확히 공개하지 않는다. 대신 플레이어가 행동의 일반적인 위험과 기대를 추측할 수 있도록 환경, 물건의 상태, 인물의 태도와 앞뒤가 맞지 않는 말 등으로 단서를 줄 수 있다.

나쁜 예시:

```text
선택 1: 꽃을 바라본다.

[피해]
[골드 획득]

피해를 받고 골드를 얻었다.
```

좋은 예시:

```text
꽃밭 한가운데에는 동전이 가득 든 주머니가 놓여 있었다. 주변 풀 위로는 작은 짐승들이 쓰러져 있었고, 달콤한 꽃향기가 숨을 막을 만큼 짙었다.

선택 1: 숨을 참고 꽃밭으로 들어가 돈주머니를 가져온다.

[피해]
[골드 획득]

일행은 옷소매로 코와 입을 막았지만 독한 꽃가루를 완전히 피하지는 못했다. 기침과 어지럼증을 견디며 중앙까지 다녀온 끝에 묵직한 돈주머니를 손에 넣었다.
```

## 반전 결과

일부 이벤트에는 선택 당시의 예상과 다른 반전 결과를 넣을 수 있다.

예시:

- 습격받는 사람들을 도우러 갔지만 그들이 행인을 유인하던 도적이어서 `[전투]`가 발생한다.
- 길에 떨어진 귀중품을 주우려 했지만 미끼 주변에 함정이 있어 `[피해]`를 받는다.
- 위험해 보이는 노인을 피했으나 실제로는 치료사였기 때문에 회복 기회를 놓치고 `[아무 일도 일어나지 않음]`으로 끝난다.
- 보수를 바라지 않고 선행했지만 도움받은 인물이 특별한 스승이어서 `[전직 이벤트]`가 열린다.
- 값비싼 물건을 주인에게 돌려주었지만 주인이 도둑으로 몰아 `[전투]` 또는 `[특수 이벤트 추가]`로 이어진다.

반전은 가끔 사용하며 모든 선택을 함정으로 만들지 않는다. 반복해서 사용하면 플레이어가 지문을 믿지 않고 항상 가장 안전한 선택만 고르게 된다.

좋은 반전에는 선택 전 지문을 다시 읽었을 때 알아볼 수 있는 작은 단서가 있다.

- 구조를 요청하는 사람들의 옷에 서로 다른 관가 문양이 섞여 있다.
- 떨어진 목걸이 주변의 흙에 여러 사람의 발자국이 숨어 있다.
- 다친 상인의 수레에는 상품보다 밧줄과 빈 자루가 더 많다.
- 친절한 인물이 자신의 이름이나 목적지를 계속 바꾸어 말한다.

단서는 반전을 확정적으로 폭로하지 않되, 주의 깊은 플레이어가 위험을 의심할 정도로 남긴다. 아무 단서도 없이 선택 결과를 뒤집어 피해나 전투를 주는 방식은 무작위 처벌처럼 보일 수 있으므로 피한다.

반전이 발생한 뒤에는 반드시 왜 예상과 달랐는지 결과 지문으로 설명한다. `사실 함정이었다`라고 한 문장으로 끝내지 말고, 누가 무엇을 노렸으며 선택 행동이 어떻게 함정을 작동시켰는지 보여준다.

## 위험과 보상의 균형

모든 선택을 단순한 정답과 오답으로 만들지 않는다.

- 안전한 선택은 보상이 적거나 아무 변화 없이 끝날 수 있다.
- 큰 보상에는 전투, 피해, 조건부 성공 같은 위험이 따를 수 있다.
- 나쁜 결과 뒤에도 유물, 골드, 후속 이벤트 같은 보상이 남을 수 있다.
- 즉시 좋은 결과가 장기적으로 가장 좋은 선택일 필요는 없다.
- 도덕적으로 선한 선택만 항상 최고의 물질 보상을 주지 않아도 된다.
- 위험한 선택이 항상 성공하거나 항상 실패하도록 고정하지 않는다.

플레이어가 결과를 알고도 감수할 만한 고민을 만드는 것이 목적이다. 다만 어떤 선택도 스토리 맥락과 관계없는 무작위 처벌처럼 느껴져서는 안 된다.

## 결과 조합 예시

사용할 수 있는 대표적인 조합은 다음과 같다.

- `[전투]` + `[유물 보상]`: 유물을 지키는 적과 싸워 승리한 뒤 획득한다.
- `[전투]` + `[골드 획득]`: 도적이나 현상범을 물리치고 보수 또는 전리품을 얻는다.
- `[피해]` + `[유물 보상]`: 함정이나 위험을 감수하고 유물을 가져온다.
- `[피해]` + `[골드 획득]`: 위험한 물건이나 돈주머니를 회수한다.
- `[피해]` + `[특수 이벤트 추가]`: 저주나 표식을 얻는 대신 후속 사건이 열린다.
- `[피해]` + `[전직 이벤트]`: 혹독한 시험을 견디고 전직 자격을 얻는다.
- `[체력 회복]` + `[특수 이벤트 추가]`: 치료를 받으며 새로운 인연이나 의뢰를 얻는다.
- `[골드 획득]` + `[특수 이벤트 추가]`: 보수를 받는 동시에 후속 의뢰가 열린다.
- `[유물 보상]` + `[특수 이벤트 추가]`: 유물이 새로운 사건의 열쇠가 된다.

모든 조합을 똑같이 자주 사용할 필요는 없다. 한 이벤트 안에서도 선택지마다 결과의 성격이 달라야 한다.

## 전체 작성 예시

```text
# 돌 틈에서 들리는 노랫소리

해가 지기 전 산길을 지나던 일행은 무너진 돌무더기 아래에서 희미한 노랫소리를 들었다. 바위틈에는 사람 하나가 겨우 들어갈 만한 틈이 있었고, 안쪽에는 금빛 장식이 희미하게 반짝였다. 주변의 마른 풀에는 검게 탄 발자국이 어지럽게 남아 있었다.

선택 1: 좁은 바위틈으로 들어가 노랫소리의 정체를 확인한다.

[전투]
[유물 보상]

일행이 안쪽의 장식을 건드리자 돌벽에 붙어 있던 검은 도마뱀들이 일제히 눈을 떴다. 도마뱀 무리를 물리킨 뒤 노랫소리가 나던 곳을 살피면, 바람이 통할 때마다 울리는 금빛 풍경 하나가 발견된다.

선택 2: 입구의 돌을 치우고 안에 갇힌 존재가 스스로 나오게 한다.

[피해]
[특수 이벤트 추가]

돌무더기를 옮기던 중 비탈이 무너져 일행은 크고 작은 상처를 입었다. 열린 틈에서는 작은 그림자 하나가 빠져나와 북쪽으로 달아났고, 그 자리에 같은 노랫소리가 나는 검은 깃털을 남겼다.

선택 3: 수상한 노랫소리에 관여하지 않고 길을 떠난다.

[아무 일도 일어나지 않음]

일행은 돌무더기를 그대로 둔 채 산길을 재촉했다. 노랫소리는 한동안 뒤를 따라왔지만 고개를 넘자 바람 소리 속으로 사라졌다.
```

## 최종 체크리스트

- [ ] 제목 다음에 요약 없이 바로 스토리가 시작되는가?
- [ ] 특정 메인 에피소드의 진행 상태를 불필요하게 전제하지 않는가?
- [ ] 선택지가 2개 또는 3개인가?
- [ ] 모든 선택지가 서로 다른 행동을 제안하는가?
- [ ] 선택지만 읽어도 행동의 목적을 알 수 있는가?
- [ ] 선택지에 시스템 결과나 정확한 보상을 직접 고지하지 않았는가?
- [ ] 행동 자체에서 자연스럽게 예상되는 결과를 억지로 숨기지 않았는가?
- [ ] 각 선택 결과가 허용된 태그 1개 또는 2개로 표시되는가?
- [ ] `[아무 일도 일어나지 않음]`을 다른 태그와 조합하지 않았는가?
- [ ] 태그 뒤에 결과를 설명하는 스토리 지문이 있는가?
- [ ] 결과가 선택 행동과 자연스럽게 연결되는가?
- [ ] 전투가 필요하지 않은 선택지와 결과도 고려했는가?
- [ ] 위험을 감수할 이유 또는 안전을 택할 이유가 있는가?
- [ ] 반전 결과가 있다면 선택 전 지문에 의심할 만한 단서가 있는가?
- [ ] 반전의 원인과 작동 방식이 결과 지문에 설명되는가?
- [ ] 보상과 손해가 이야기 속 원인에서 발생하는가?
- [ ] 사건이 자체적으로 끝나거나 후속 이벤트가 명확히 표시되는가?
- [ ] `[특수 이벤트 추가]`가 있다면 연계 스토리가 같은 파일에 작성되어 있는가?
- [ ] 연계 스토리가 별도의 `# 제목`으로 구분되어 있는가?
- [ ] 결과 지문에 후속 스토리가 열리는 이유와 연결 단서가 있는가?
