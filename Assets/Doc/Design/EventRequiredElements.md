# 이벤트 제작 필수 요소

- formatId: `design.rule.event_required_elements`
- version: `2`

## 목적

Act 번호와 생성할 이벤트 개수, 선택적인 전직 이벤트와 특이사항을 입력받아 해당 Act에 어울리는 단독 이벤트 스토리를 자동으로 구성하기 위한 기준과 입력 양식을 정리한다.

이벤트를 만들 때 작성 규칙은 항상 `Assets/Doc/Design/EventFormat.md`를 참조한다.

사용자에게는 `생성할 Act 번호`, `생성할 이벤트 개수`, 선택적인 `전직 이벤트`와 `특이사항`을 입력받는다. 이벤트 제목, 장소, 사건, 등장인물, 선택지, 결과, 반전과 연계 이벤트 여부는 전직 이벤트 또는 특이사항에 별도 지시가 없으면 질문하지 않고 제작자가 자동으로 구성한다.

Act 번호는 두 자리 숫자로 맞춰 다음 경로를 자동 조합한다.

```text
Assets/Doc/Story/Act{Act번호}
```

예시:

```text
생성할 Act 번호: 1
자동 Act 스토리 경로: Assets/Doc/Story/Act01
```

자동 조합한 Act 폴더의 배경, 전체 줄거리, Chapter 요약과 Episode 파일을 확인하고 세계관, 지역, 분위기와 서술 문체를 맞춘다. 이벤트는 해당 Act의 에피소드 사이에 들어갈 수 있어야 하지만, 특정 Episode의 직전이나 직후 상태에 의존하지 않도록 작성한다.

완성된 이벤트 파일은 Act 번호를 두 자리로 맞춘 전용 폴더에 생성한다.

```text
Assets/Doc/Event/Act{Act번호}
```

예시:

```text
생성할 Act 번호: 1
자동 이벤트 생성 경로: Assets/Doc/Event/Act01

생성할 Act 번호: 2
자동 이벤트 생성 경로: Assets/Doc/Event/Act02
```

해당 Act 폴더가 없다면 새로 만든 뒤 이벤트 파일을 생성한다. 이벤트 파일을 `Assets/Doc/Event` 바로 아래에 만들거나 다른 Act 폴더에 섞지 않는다.

이벤트 파일은 아래 형식의 이름을 사용한다.

```text
{두 자리 순번}_{이벤트_제목}.md
```

예시:

```text
Assets/Doc/Event/Act01/01_비를_기다리는_돌탑.md
Assets/Doc/Event/Act01/02_빈_집의_저녁밥.md
```

파일 순번은 생성 대상인 `Assets/Doc/Event/ActXX` 폴더 안의 기존 이벤트 파일을 확인한 뒤 겹치지 않는 다음 번호를 사용한다. 다른 Act 폴더의 순번은 고려하지 않으며, 각 Act는 `01`부터 독립적으로 번호를 사용할 수 있다. 제목의 띄어쓰기는 밑줄로 바꾸고, 본문 첫 줄에는 실제 이벤트 제목을 `# 제목` 형식으로 작성한다.

이벤트 파일 맨 위에는 줄거리 요약, 목적, 삽입 위치, 보상 목록을 적지 않는다. 제목 다음부터 바로 스토리를 시작한다.

## 자동 구성 요소

아래 항목은 사용자에게 입력받지 않고, Act 스토리와 `EventFormat.md` 규칙을 기준으로 제작자가 정한다.

- 이벤트 제목
- 사건이 발생하는 장소
- 처음 발견하는 상황 또는 문제
- 주요 등장인물 또는 대상
- 플레이어가 선택할 수 있는 행동 2개 또는 3개
- 각 선택에 따른 결과 태그 1개 또는 2개
- 각 결과가 발생하는 이야기상 이유
- 선택 전에 위험이나 기대를 짐작할 수 있는 지문
- 반전 결과의 유무
- 반전이 있다면 선택 전에 배치할 작은 단서
- `[특수 이벤트 추가]` 사용 여부
- 특수 이벤트가 있다면 같은 파일에 작성할 연계 스토리 제목과 내용
- 특이사항

결과 태그는 다음 목록에서만 선택한다.

- `[전투]`
- `[체력 회복]`
- `[피해]`
- `[유물 보상]`
- `[골드 획득]`
- `[특수 이벤트 추가]`
- `[전직 이벤트]`
- `[아무 일도 일어나지 않음]`

각 선택에는 결과 태그를 최대 2개까지 사용할 수 있다. `[아무 일도 일어나지 않음]`은 다른 결과와 조합하지 않는다.

실제 이벤트의 선택지 문구에는 결과 태그, 보상 종류, 정확한 보상량 또는 발생할 전투를 직접 고지하지 않는다.

## 특수 이벤트 필수 요소

`[특수 이벤트 추가]`를 사용하는 경우 아래 항목을 추가로 정한다.

- 연계 스토리 제목
- 연계 스토리가 열리는 이유
- 최초 이벤트 결과에 남길 연결 단서
- 연계 스토리의 시작 상황
- 연계 스토리의 선택지 2개 또는 3개
- 연계 스토리의 선택별 결과

연계 스토리는 별도 파일로 만들지 않는다. 최초 이벤트와 같은 파일 아래에 새로운 `# 제목`을 붙여 이어서 작성한다.

```text
# 최초 이벤트 제목

최초 이벤트 스토리...

# 연계 스토리 제목

연계 이벤트 스토리...
```

연계 스토리가 작성되지 않았다면 최초 이벤트에서 `[특수 이벤트 추가]`를 사용할 수 없다.

## 입력 양식

```text
생성할 Act 번호:
생성할 이벤트 개수:
전직 이벤트: 없으면 비워 둠
특이사항: 없으면 비워 둠
```

이 네 항목 외의 입력은 추가로 요구하지 않는다. `전직 이벤트`와 `특이사항`은 필수가 아니다. 전직 이벤트는 `{현재 직업} -> {전직 직업}` 형식으로 입력하며, 여러 개라면 쉼표로 구분한다. 값이 비어 있으면 `[전직 이벤트]`를 사용하지 않는다. 전직 이벤트의 실제 구성과 작성 방식은 `EventFormat.md`를 따른다. 특이사항이 비어 있으면 제작자가 전직을 제외한 전체 구성을 결정한다. 특이사항에 포함할 소재, 제외할 소재, 원하는 분위기, 반드시 포함하거나 제외할 결과, 특정 장소, 반전 또는 특수 이벤트 요구가 적혀 있다면 `EventFormat.md`와 충돌하지 않는 범위에서 우선 반영한다.

입력받은 개수만큼 서로 다른 이벤트 파일을 생성하며, 기존 이벤트를 개수에 포함하거나 덮어쓰지 않는다. 필요한 모든 구성은 `EventFormat.md`, 해당 Act의 스토리, `Assets/Doc/Event/ActXX`의 기존 이벤트를 기준으로 결정한다. 결과는 선택지 문구에 직접 노출하지 않고, 선택 이후의 결과 지문에서만 결과 태그와 내용을 보여준다.
